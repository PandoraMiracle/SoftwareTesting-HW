## 1. Fundamental Test Process (Main Activities)

### 1.1 Test Planning
- Define objectives, approach, scope; test plan, test strategy, estimation, environment requirements
- Entry criteria and exit criteria (defined at planning stage)
- Roles and responsibilities; test tools selection

### 1.2 Test Monitoring and Control
- Track progress, coverage, defect metrics; re-prioritize; apply entry/exit criteria decisions

### 1.3 Test Analysis
- Identify **what** to test; define **test conditions** from requirements, user stories, risks

### 1.4 Test Design
- Transform test conditions into **test cases**; test data; traceability

### 1.5 Test Implementation
- Develop test procedures, suites, environment setup, test data loading

### 1.6 Test Execution
- Run tests; log results; report defects; confirmation and regression testing

### 1.7 Test Completion
- Check exit criteria; archive testware; test summary report; lessons learned

## 2. Test Levels
- Component (Unit) Testing
- Integration Testing — strategies: Big-bang, Top-down, Bottom-up, **Functional incremental / sandwich integration**
- System Testing
- Acceptance Testing — UAT, OAT, contractual, Alpha, Beta

## 3. Test Types
- Functional Testing
- Non-Functional Testing — ISO 25010: **Functional suitability**, Performance efficiency, Compatibility, Usability, Reliability, Security, Maintainability, Portability
- Structural (White-Box) Testing
- Change-Related Testing — Confirmation testing (re-testing), Regression testing

## 4. Static vs Dynamic Testing

### Static Testing
- Reviews: informal, walkthrough, technical review, inspection, management review, **audit**
- Automated static analysis (SAST)

### Dynamic Testing
- Requires executing software; manual/automated/exploratory execution

## 5. Testware
- Test conditions (from analysis) → Test cases (from design) → Test procedures (from implementation)

## 6. Entry / Exit Criteria
- Entry: test plan approved, environment ready, previous level exit met
- Exit: coverage thresholds, defect thresholds, test summary approved

## 7. Defect Lifecycle
- New → Assigned → Fixed/Resolved → Re-test → Verified/Closed; Deferred; Rejected

## 8. Roles
- Test Manager — planning, monitoring, completion
- Tester — execution, defect reporting
- Test Analyst — analysis and design
